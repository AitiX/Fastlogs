# FastLogs - логи игры в одну ссылку (гайд команды)

> Версия: **v0.3.0** (2026-06-13). Репозиторий: `https://github.com/AitiX/Fastlogs` (private). Источники правды: `docs/TEAM-GUIDE.md` (полная версия с примерами) и `CONTRACT.md` (контракт API).

## Что это

FastLogs - кросс-движковый диагностический инструмент. Из запущенной игры одним жестом отправляешь рантайм-логи плюс полный срез устройства (и опц. скриншот) на свой сервер и получаешь короткую ссылку для просмотра в браузере. Ссылки складываются в каталог **Server -> Project -> version -> Log**. Движки: **Unity** (C#, UPM-пакет) и **GameMaker** (GML). Платформы: WebGL и веб-порталы, iOS, Android, Standalone (Windows / macOS / Linux). На консолях в релизе инструмент полностью вырезается (нет HTTP/оверлеев, нет конфликтов на сертификации).

Зачем это нужно: в мобильных, WebGL и консольных билдах у разработчика нет доступа к консоли движка, а тестеру неудобно описывать баг и пересылать логи. Тестер ловит баг, одной кнопкой отправляет лог и кидает короткую ссылку разработчику. Краши улетают сами и не теряются. Один сервер обслуживает все игры и движки.

## Возможности

- Отправка одним жестом: рантайм-логи + полный срез устройства + опц. скриншот -> короткая ссылка на вьюер в браузере.
- Кросс-движковость: один сервер и один контракт API для Unity (C#) и GameMaker (GML).
- Контекст - произвольные пары ключ -> значение (что делал игрок, уровень, стейт), едут с каждым отчётом; во вьюере секция **Context**.
- Breadcrumbs - катящийся след последних событий (кольцо до 100); во вьюере таймлайн **Breadcrumbs**.
- Приватность по умолчанию: PII в логах, контексте и крошках режется в `[redacted]`, чувствительные поля и сырой IP не шлются.
- Краш захватывается всегда: необработанный краш сначала пишется в дисковый outbox (до предохранителей), поэтому не теряется и доедет на следующем запуске.
- Отправка файла/папки из кода: `SendFile` / `SendFolder` (папка зипуется в один `.zip` на клиенте) -> короткая ссылка с кнопкой Download во вьюере. Типичный кейс - выгрузить файл сейва с устройства разработчику. Можно приложить файл к обычному отчёту. Бинарь идёт байт-в-байт (PII-скраб к нему не применяется), кап по размеру `MAX_FILE_BYTES` (по умолчанию 25 MB).
- Каталог логов (Server -> Project -> version -> Log) под командным viewer-токеном; ретеншн с возможностью Pin.
- В релизе и на консолях инструмент вырезается целиком (накладные расходы 0, без конфликтов сертификации).

Подробности - в разделах ниже.

## Для QA (тестировщикам)

Всё, что нужно тестеру. Настраивать ничего не надо, инструмент уже встроен в билд.

> Нужен билд с включённым FastLogs: Dev-сборка или релиз с дефайном `LOGSHARE_FORCE_ENABLED`. Для Terraformers - это TestFlight-сборки с этим дефайном.

### Как снять и отправить лог

1. Поймал баг.
2. Открой оверлей: **3 быстрых тапа в правый верхний угол экрана** (или `F8`). В оверлее видны счётчики E/W/L, тоггл «Скриншот» и поле комментария.
3. Опционально: включи скриншот, впиши заголовок и комментарий (что делал, как воспроизвести).
4. Нажми **Отправить**.
5. Короткая ссылка авто-копируется в буфер устройства (в Unity дополнительно показывается QR ссылки - удобно перекинуть на другое устройство).
6. Кинь ссылку разработчику. Ссылка работает без токена - открывается у любого по ней.

### Что увидит разработчик

По твоей ссылке откроется вьюер, где видно:

- Логи с фильтрами **E/W/L** (ошибки / предупреждения / обычные).
- Полный срез устройства (модель, ОС, версия игры и т.д.).
- Скриншот (если включал).
- Контекст - что делал игрок (уровень, режим, стейт).
- Breadcrumbs - хронологию последних событий перед багом.
- Твой комментарий и заголовок.

При сбое сети отчёт сам повторяется, пока не пройдёт. Краши улетают сами и не теряются - если игра упала жёстко, отчёт доедет на следующем запуске. Настраивать тестеру ничего не надо.

## Для разработчиков

Архитектура, интеграция в Unity и GameMaker, контекст и breadcrumbs, краш-надёжность, приватность, гейтинг для релиза и консолей, ретеншн, просмотр логов, подключение проекта и CI.

### Архитектура

|| Слой || Что делает ||
| Клиент (Unity C# / GameMaker GML) | Собирает кольцо логов + срез устройства + опц. скриншот + контекст + breadcrumbs, режет PII, шлёт `POST /api/logs` (JSON по `CONTRACT.md`) |
| Сервер (Node + SQLite за nginx) | Принимает, хранит, отдаёт короткую ссылку, кладёт в каталог, асинхронно форвардит в sinks |
| Просмотр | Вьюер `GET /<id>`, командный каталог `GET /browse` |

```
Unity / GameMaker --POST /api/logs--> Сервер (SQLite + blobs)
   -> короткая ссылка https://<host>/<id>   (вьюер)
   -> каталог /browse                        (Server -> Project -> version -> Log)
   -> опц. sinks                             (Slack / Discord / Confluence / webhook)
```

### Быстрый старт: Unity

Версии Unity: **6000.1 (Unity 6)** и **2022.3 LTS**.

**1. Установка** (Git URL) - в `Packages/manifest.json`:

```json
{
  "dependencies": {
    "com.playjoy.fastlogs": "https://github.com/AitiX/Fastlogs.git?path=/unity#main"
  }
}
```

**2. Конфиг**: `Tools > PlayJoy > FastLogs > Create Config Asset` (создаёт `Assets/Resources/FastLogsConfig.asset`).

**3. Заполнить секцию Server**:

|| Поле || Значение ||
| `EndpointUrl` | `https://logs.example.com/api/logs` (обязательно; на iOS только https) |
| `AppId` | `[a-z0-9_-]{2,32}` - Project в каталоге |
| `Token` | Bearer-токен ингеста (опц.) |
| `RetentionDaysOverride` | дни хранения; `0` = серверный дефолт |

**4. Код** (запись по умолчанию ВЫКЛ):

```csharp
using PlayJoy.FastLogs;

FastLogs.Init();              // грузит Resources/FastLogsConfig; идемпотентен
FastLogs.StartRecording();    // включить запись (или AutoStartRecording в конфиге)

FastLogs.Log("Checkpoint");
FastLogs.Warn("Low memory");
FastLogs.Error("Spawner failed");

using (FastLogs.RecordScope()) { /* пишется только этот участок */ }

// Контекст и breadcrumbs - едут с каждым отчётом:
FastLogs.SetContext("level", "3");          // пара ключ -> значение (null-значение удаляет ключ)
FastLogs.SetContext("playerId", "abc");
FastLogs.Breadcrumb("Player opened shop");  // событие в катящийся буфер (опц. level)
FastLogs.ClearContext();                     // при смене сцены/сессии

// Отправка:
FastLogs.Send();              // быстрая отправка без UI (хоткей/жест/из кода)
var r = await FastLogs.SendAsync(includeScreenshot: true, title: "Crash on load", comment: "Открыл уровень 3, зависло");
if (r.Success) GUIUtility.systemCopyBuffer = r.Url;

FastLogs.ToggleOverlay();     // оверлей: F8 по умолчанию
```

В отчёт идут: логи (кольцо + ручные), полный device-срез, контекст, breadcrumbs, опц. скриншот, комментарий и имя тестера. После ответа оверлей показывает ссылку, кнопки «Copy»/«Open» и **QR ссылки** (для перекидывания на другое устройство). Готовый пример - сэмпл **Basic Usage** (Package Manager -> Samples).

Хоткеи (настраиваются в секции Trigger конфига): оверлей - `F8`; быстрая отправка без оверлея - `F9` (по умолчанию выключена). На мобильных - мульти-тач по углу/жест (по умолчанию выключены).

### Быстрый старт: GameMaker

> Упакованного `.yymps` пока нет (планируется). Переносите скрипты `scr_fastlogs_*` и объект `obj_fastlogs_controller` из `gamemaker/` в свой проект.

**1. Перенести** в проект все `scr_fastlogs_*` и `obj_fastlogs_controller`; завести build-конфиг с именем `debug` (имя обязано совпадать с `.yyp`; в нём `FASTLOGS_ENABLED = true`).

**2. Настроить `scr_fastlogs_config`** (или переопределить макросы в своём проекте):

```gml
#macro FASTLOGS_ENDPOINT    "https://logs.example.com/api/logs"
#macro FASTLOGS_APP_ID      "mygame"
#macro FASTLOGS_TOKEN       ""
#macro FASTLOGS_APP_VERSION ""
```

**3. Контроллер** `obj_fastlogs_controller` (persistent) в первой комнате, либо `fastlogs_init()` в бутстрапе.

**4. Логирование** - одной строкой в существующий `trace()`:

```gml
function trace(_msg) {
    show_debug_message(_msg);
    flog(_msg);              // весь существующий лог идёт в FastLogs
}
```

**5. Контекст, breadcrumbs, запись и отправка** (запись по умолчанию ВЫКЛ):

```gml
fastlogs_record_start();                 // включить запись

fastlogs_set_context("level", "3");      // пара ключ -> значение
fastlogs_set_context("playerId", "abc");
fastlogs_breadcrumb("Player opened shop"); // событие в катящийся буфер (опц. level)
fastlogs_clear_context();                 // при смене комнаты/сессии

fastlogs_set_screenshot(true);           // опц. скриншот
fastlogs_send({ title: "Crash" });       // -> bool
fastlogs_quick_send();                   // быстрая отправка без UI (хоткей/жест/из кода)
fastlogs_toggle();                       // оверлей (vk_f8)
var url = fastlogs_last_url();
```

Хоткеи: оверлей - `vk_f8`; быстрая отправка - `vk_f9`. Оверлей в GameMaker рисуется примитивами (без спрайтов); QR в нём не выводится.

### Контекст и breadcrumbs (для триажа)

Эти два поля резко ускоряют разбор бага: видно, что делал игрок и что происходило прямо перед отчётом, без угадывания.

**Контекст** - произвольные пары ключ -> значение (строки), снимок состояния, который едет с каждым отчётом. Типичное: текущий уровень, режим, id игрока, версия контента, фича-флаги. Ставится один раз и обновляется по ходу игры. Во вьюере - сворачиваемая секция **Context**.

|| Действие || Unity || GameMaker ||
| Задать пару | `FastLogs.SetContext(key, value)` | `fastlogs_set_context(key, value)` |
| Удалить ключ | `FastLogs.SetContext(key, null)` | `fastlogs_remove_context(key)` |
| Очистить всё | `FastLogs.ClearContext()` | `fastlogs_clear_context()` |

**Breadcrumbs** (хлебные крошки) - катящийся след последних событий: кольцо из последних ~100 записей, старые вытесняются. Кладёшь короткое событие в ключевых точках («открыл магазин», «начал бой», «нажал Continue»), и в отчёте получаешь хронологию пути к багу. Во вьюере - секция/таймлайн **Breadcrumbs**.

|| Действие || Unity || GameMaker ||
| Добавить крошку | `FastLogs.Breadcrumb(message, level = Log)` | `fastlogs_breadcrumb(message, [level])` |

Уровень крошки нормализуется к `info` / `warn` / `error`. Контракт поля: `context` - объект `string->string`; `breadcrumbs` - массив `{ t, m, lvl }`. Сервер режет по капам (`context` ~4 KB суммарно / ключ <= 64 / значение <= 512; `breadcrumbs` 100 шт / ~16 KB), приводит значения к строкам и игнорирует невалидный тип, не валя запрос. Пустые `context`/`breadcrumbs` не отправляются. Запись крошки и контекста дешёвая (O(1), без аллокаций на кадр); сериализация и PII-чистка делаются разово при сборке отчёта.

### Отправка файла / папки (SendFile / SendFolder)

Кроме лог-отчёта из кода можно отправить произвольный **файл** или **папку** и получить такую же короткую ссылку - с кнопкой **Download** во вьюере. Папка зипуется **на клиенте** в один `.zip`. Типичный кейс: выгрузить файл сейва с устройства разработчику. Гейтинг общий с FastLogs (в ритейле и на консолях вырезано). Бинарь идёт **байт-в-байт**: PII-скраб к нему не применяется, единственная защита - размерный кап `MAX_FILE_BYTES` (по умолчанию 25 MB по распакованному размеру).

**Unity.** Fire-and-forget `void`-перегрузки `[Conditional]` (стрипаются в ритейле); awaitable перегрузки возвращают `FlogTask<FileUploadResultDto>` (`Success`, `Url`, `DownloadUrl`, `Id`, `ExpiresAt`, ...). На **WebGL** доступа к ФС по пути нет - там работают только `byte[]`-перегрузки, а path-перегрузки честно возвращают Fail.

```csharp
using PlayJoy.FastLogs;

FastLogs.SendFile(Application.persistentDataPath + "/save_slot_3.sav");  // fire-and-forget
FastLogs.SendFolder(Application.persistentDataPath + "/saves");          // папка -> один .zip

var r = await FastLogs.SendFileAsync(savePath, title: "Save before crash");
if (r.Success) GUIUtility.systemCopyBuffer = r.DownloadUrl;              // прямая ссылка на скачивание

await FastLogs.SendFileAsync(bytes, "save_slot_3.sav");                  // WebGL-safe (по byte[])
await FastLogs.SendFolderAsync(dirPath);                                 // зип папки
await FastLogs.SendFilesAsync(new[] { pathA, pathB });                   // зип нескольких файлов

FastLogs.AttachFile(savePath);                                           // приложить к СЛЕДУЮЩЕМУ отчёту
FastLogs.Send();
```

**GameMaker.** Возвращают `bool` (поставлен ли запрос). `opts` (struct, опц.): `title`, `logId`, `groupId`, `mime`, `kind` (автодефолт `file` / `folder`), `retentionDays`, `name` (имя в архиве/каталоге), `onDone` (`function(result)` - колбэк).

```gml
fastlogs_send_file(filename_dir(game_save_id) + "save_slot_3.sav");
fastlogs_send_folder(working_directory, { title: "Saves" });   // папка -> один .zip
fastlogs_send_files([path_a, path_b], { title: "Two saves" }); // несколько файлов -> один .zip
```

**Сервер.** Отдельный эндпоинт `POST /api/files` (JSON + base64, **не** часть лог-отчёта), у него отдельный больший лимит тела. Скачивание - `GET /files/<id>/download` (блоб с `Content-Disposition: attachment`); просмотр - `GET /files/<id>` (лёгкий HTML с Download). Файл, отправленный с `logId`, дополнительно появляется в панели **Attachments** вьюера лога. Полное тело и коды ошибок - в `CONTRACT.md` (§1b).

### Краш-надёжность (захват всегда + досыл)

Краши устроены так, чтобы не терять необработанные исключения, даже когда процесс падает целиком.

Каждый необработанный краш сначала пишется в дисковый outbox (готовое JSON-тело отчёта), и только потом инструмент пытается его отправить - поэтому даже хард-краш, убивший процесс до завершения HTTP, не теряется. Незавершённые отчёты досылаются при простое в этой же сессии или на следующем запуске, когда `Init` сканирует очередь; тестеру делать ничего не надо. Цикл одинаковых крашей не плодит файлы: один и тот же стек подавляется в окне (по умолчанию около окна троттла), а не навсегда, и если краш повторится позже, он доставится снова.

Предохранители касаются только темпа аплоада, не захвата: троттл (по умолчанию 30 с между авто-отправками), кап на сессию (по умолчанию 10 авто-доставок) и блокировка во время отправки ограничивают доставку, но не запись в outbox. Постоянная 4xx (например битый или неавторизованный отчёт) убирает файл из outbox как poison-pill, чтобы он не висел вечно.

Авто-отправка при необработанном исключении включена по умолчанию в dev (Unity `AutoSendOnException = true`, GameMaker `FASTLOGS_AUTOSEND_ON_EXCEPTION = true`); скриншот к крашу не прикладывается. В релизе и на консолях инструмент вырезан целиком, поэтому краши там не шлются by design. Ручные `Send` / `quick_send` не троттлятся - лимиты выше касаются только авто-отправки по крашу.

### Приватность по умолчанию

Из коробки наружу уходит минимум, лишнего не уезжает.

Сырой IP тестера клиент не отправляет вовсе. Для rate-limit сервер берёт IP только из соединения и хранит как `соль + sha256` (см. `IP_SALT`), а не в открытом виде, так что адрес не восстановить. Чувствительные поля устройства, URL и идентификаторы (имя устройства, ссылки, идентификаторы) - под флагом, выключенным по умолчанию: Unity `IncludeSensitive = false`, GameMaker `FASTLOGS_INCLUDE_SENSITIVE = false`.

PII-чистка работает best-effort. Перед отправкой и перед записью краша в дисковый outbox в тексте логов, значениях контекста и текстах breadcrumbs редактируются в `[redacted]`: email, IPv4, IPv6, Bearer/Authorization-токены и длинные цифровые последовательности (9+ цифр). Включено по умолчанию (Unity `ScrubPii = true`, GameMaker `FASTLOGS_SCRUB_PII = true`), разово на отправке или краше, не на горячем пути логирования.

Настройка идёт из панели настроек оверлея или конфига. Отправка чувствительных полей включается тогглом *include sensitive* (`IncludeSensitive` / `FASTLOGS_INCLUDE_SENSITIVE`), PII-чистка выключается тогглом *scrub PII* (`ScrubPii` / `FASTLOGS_SCRUB_PII`). Набор паттернов расширяется: Unity `PiiScrubber.AddPattern(...)` до первой отправки, GameMaker `fastlogs_redact_rules_set(...)`.

Чистка по регуляркам может перередактировать (длинные числа или таймстемпы попадут под правило 9+ цифр) - осознанный компромисс в пользу приватности: лучше лишний `[redacted]`, чем утёкший токен. Как единственную меру полагаться на неё не стоит. Логи и скриншоты всё равно могут содержать персональные данные; скриншот по умолчанию выключен и шлётся только по явному запросу. Доступ непубличный: короткая непредсказуемая ссылка, единый `404` на невалидный id, каталог под viewer-токеном. Секреты (токены, ключи деплоя) - никогда в репозиторий.

### Консоли и релиз (гейтинг)

В ритейле и на консолях инструмента **нет вообще** (накладные расходы = 0, нет конфликтов сертификации).

|| Движок || Механизм ||
| Unity | `#if UNITY_EDITOR \|\| DEVELOPMENT_BUILD` (+ `LOGSHARE_FORCE_ENABLED` для не-консольного release). Void-методы `[Conditional]` - стрипаются вместе с вызовами. PS4/PS5/GameCore/Switch жёстко заблокированы. **Публичный `FastLogs.*` компилируется всегда** - игра билдится везде. |
| GameMaker | `#macro Default: FASTLOGS_ENABLED false` / `#macro debug: FASTLOGS_ENABLED true`. В релизе контроллер не создаётся, все функции - no-op, нет `http_request`/`screen_save`. |

Включить в non-console release: Unity - `Build Defines Helper` (`LOGSHARE_FORCE_ENABLED`) + `EnableInRelease = true`; GameMaker - собирать конфигом `debug`.

### Хранение логов: ретеншн и Pin

По умолчанию лог живёт 30 дней, потом его удаляет фоновый sweeper.

Чтобы лог жил дольше, открой его ссылку и нажми кнопку `Pin` во вьюере. Закреплённый лог не удаляется, пока закрепление не снято. Кнопка Pin доступна прямо по ссылке, так что тестеру не нужны права; снять закрепление (unpin) или удалить лог может только админ (admin-токен).

Нужен конкретный срок - задай его при отправке через `retentionDays` (Unity `RetentionDaysOverride`, GameMaker `FASTLOGS_RETENTION_DAYS`). Сервер делает clamp в диапазон `[1, макс]`, максимум по умолчанию - 365 дней.

|| Сценарий || Что сделать ||
| Лог нужен «навсегда» (баг в работе) | Открыть ссылку -> нажать **Pin** |
| Лог нужен на N дней | Задать `retentionDays` при отправке (1..365) |
| Лог не нужен | Ничего не делать - удалится через 30 дней |
| Снять Pin / удалить досрочно | Только админ (admin-токен) |

### Как смотреть логи

**Вьюер** `GET /<id>`: фильтр E/W/L, поиск, raw/pretty, copy, сворачивание стектрейсов, сворачиваемые device-группы, скриншот, кнопка **Pin**, секция **Context** (пары ключ->значение) и таймлайн **Breadcrumbs**. Доп.: `/<id>/raw` (сырой лог; `?download=1` отдаёт файлом), `/<id>/screenshot` (PNG).

**Каталог** `GET /browse` (под team viewer-токеном):

|| Уровень || Endpoint || Показывает ||
| Project | `/browse` | проекты (`appId` + имя) |
| version | `/browse/<appId>` | версии с количеством записей |
| Log | `/browse/<appId>/<version>` | записи: id, title, time, platform, counts, pinned |

Маппинг: **Server** = инстанс сервера · **Project** = `appId` · **version** = `appVersion` · **Log** = `id`.

Каталог `/browse` закрыт общим viewer-токеном. Чтобы открыть каталог, добавь `?token=ТОКЕН` в конец URL. Токен один на всю команду (лежит в 1Password / у админа). Сами Send-ссылки токен не требуют - они открываются у любого. Пример:

```text
https://fastlogs.playjoystudios.com/browse/terraformers?token=ТОКЕН
```

### Подключить новый проект + CI (Terraformers / TeamCity)

**Регистрация проекта на сервере.** Два пути:

- **Открытый ingest** (без токена) - выполнить на сервере:

```text
docker exec -w /app fastlogs node scripts/add-app.js <appId> "<Name>" --no-token
```

- **Общий токен + self-onboard** - задать `TEAM_INGEST_TOKEN` и `ALLOW_AUTO_REGISTER=1`; тогда проект сам регистрируется при первом Send.

Проект **terraformers** уже подключён как открытый.

**CI (TeamCity).** Шаг Build: runner-тип **python-runner**, скрипт `build.py`. Дефайны передаются в параметр `args` как `--extradefines=...`. Несколько дефайнов **обязательно в кавычках**, иначе shell режет строку по `;`:

```text
"--extradefines=LOGSHARE_FORCE_ENABLED;ENABLE_SRDEBUG_IN_RELEASE"
```

Триггерить сборку **без** `branchName` (используется дефолтная ветка `porting`). Указание `branchName=porting` ломает VCS-матч и метит билд как FAILURE.

### FAQ

|| Симптом || Что делать ||
| Не пришла ссылка | Проверь endpoint, включена ли запись (ВЫКЛ по умолчанию), вызван ли Init, не вырезан ли клиент в этом билде. Unity: `OnUploaded` / `result.Error`. GM: `http_status`. |
| Лог пустой | Включи запись: `StartRecording` / `fastlogs_record_start` / автостарт / `RecordScope`. |
| В логе всюду `[redacted]` | Это PII-чистка (включена по умолчанию). Если режет лишнее (длинные числа/таймстемпы) - временно выключи тоггл *scrub PII* или сузь паттерны. |
| Нет полей устройства (модель/URL) | Они под флагом `IncludeSensitive` (выключен по умолчанию). Включи осознанно. |
| Краш не доехал сразу | Нормально: хард-краш доедет на следующем запуске (досыл из outbox). Проверь, что `Init` вызывается на старте. |
| Лог пропал через месяц | Дефолтный ретеншн 30 дней. Нужен дольше - открой ссылку и нажми **Pin** заранее. |
| CORS на WebGL | Тело шлётся plain, copy/open из клика (делается автоматически). Проверь `CORS_ALLOW_ORIGIN` на сервере. |
| 401 / 403 | 401 - не передан токен (впиши `Token` / `FASTLOGS_TOKEN`). 403 - неверный токен или `appId` не зарегистрирован: попроси админа. |
| 413 / лог обрезан | Превышены лимиты (тело лога ~8 MB, скриншот ~2 MB, лог ~20 MB). Уменьши `MaxLogTextBytes` / скриншот. Файл/папка - свой кап `MAX_FILE_BYTES` (~25 MB) и свой больший лимит тела `/api/files`; за nginx проверь `client_max_body_size` (деф. 10m режет крупные файлы). |
| iOS не шлёт | Endpoint обязан быть `https://` (ATS). |
