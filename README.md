# FastLogs

Кросс-движковый инструмент: из запущенной игры одним жестом отправить рантайм-логи (+ полный срез устройства и опц. скриншот) на свой сервер и получить **короткую ссылку** для просмотра в браузере. Все ссылки складываются в каталог `Server -> Project -> version -> Log`.

## Зачем это и какую проблему решает

В мобильных, WebGL и консольных билдах у разработчика нет доступа к консоли движка, а тестеру неудобно описывать баг и пересылать логи. Раньше логи шарили через сторонний PrivateBin (живёт час-неделю, без структуры и без среза устройства). FastLogs закрывает это:

- Тестер ловит баг и одной кнопкой/жестом отправляет лог -> получает короткую ссылку (в Unity-оверлее ещё и QR этой ссылки, чтобы снять камерой другого устройства) и кидает её разработчику.
- Разработчик открывает ссылку в браузере: консоль с фильтрами E/W/L и поиском, полный срез устройства, скриншот, комментарий и имя тестера.
- Краши улетают сами (авто-отправка при необработанном исключении) и не теряются - сначала пишутся в дисковый outbox и доедут на следующем запуске; собираются в командном каталоге даже без участия тестера.
- Это переиспользуемый внутренний сервис: один сервер на все игры и движки, ссылки хранятся структурированно, срок хранения настраивается.

## Движки и платформы

- Движки: **Unity** (C#, UPM-пакет) и **GameMaker** (GML-пакет). Единый JSON-контракт - под новый движок добавить клиент несложно.
- Платформы: WebGL и веб-порталы (CrazyGames / GameDistribution / Yandex / Playgama), iOS, Android, Windows / macOS / Linux.
- Консоли: в релизных билдах инструмент полностью вырезается (нет HTTP/оверлеев -> нет конфликтов на сертификации). Публичный API при этом компилируется всегда, так что код игры собирается везде.

## Возможности

- Захват логов в кольцевой буфер + запись на диск (персистентно, переживает рестарт/краш), по команде.
- Полный диагностический срез устройства: system / graphics / display / application / runtime / memory / network / build / web.
- Опциональный скриншот (даунскейл, по умолчанию выкл).
- Контекст: произвольные пары ключ->значение (что делал игрок / уровень / стейт), едут с каждым отчётом; во вьюере секция **Context**.
- Breadcrumbs: катящийся след последних событий (кольцо до 100), во вьюере секция/таймлайн **Breadcrumbs**.
- Комментарий тестера и имя тестера в отчёте.
- Оверлей с независимой панелью настроек (доступен из коробки в Editor / Development-билдах).
- Отправка: из оверлея, по хоткею/жесту или из кода без UI (`FastLogs.Send()` / `fastlogs_quick_send()`).
- Статус-тост (Отправка -> Готово + ссылка / Ошибка) и авто-копирование ссылки в буфер устройства (ПК/телефон). После успешной отправки Unity-оверлей показывает ссылку с кнопками Copy/Open и QR этой ссылки (для перекидывания на другое устройство); в GameMaker QR нет.
- Краш захватывается всегда: каждый необработанный краш сначала пишется в дисковый outbox (до предохранителей), поэтому не теряется; дедуп, чтобы цикл крашей не плодил файлы; доставка при первой возможности или на следующем запуске (даже хард-краш, убивший процесс, доедет). Троттл / кап-на-сессию / блокировка-во-время-отправки ограничивают только темп аплоада, не захват; постоянная 4xx убирает файл из outbox (poison-pill). Авто-отправка при необработанном исключении включена по умолчанию в dev.
- Приватность по умолчанию: PII в логах/контексте/крошках режется, чувствительные поля и сырой IP не шлются (см. раздел ниже).
- Ретрай до успеха при сетевых сбоях (интервал настраивается).
- Сервер: вьюер (фильтры / поиск / raw / скриншот / pin), каталог `/browse`, форвардинг ссылок (sinks: webhook / Slack / Discord / Confluence), ретеншн + авто-уборка (sweeper), мульти-игра (per-app токены или общий team-токен с авто-регистрацией).
- Производительность: без аллокаций на горячем пути (на каждый лог), запись на диск батчингом, ноль работы при закрытом оверлее, ретрай на таймере.

## Как работает

```
Unity / GameMaker клиент
  -> POST /api/logs  (логи + срез устройства + опц. скриншот / комментарий)
     -> сервер (Node + SQLite за nginx)
        -> короткая ссылка  https://<host>/<id>   (вьюер)
        -> каталог  /browse                        (Server -> Project -> version -> Log)
        -> опц. sinks                              (Slack / Discord / Confluence / webhook)
```

## Быстрый старт

### Unity
В `Packages/manifest.json`:
```json
"com.playjoy.fastlogs": "https://github.com/AitiX/Fastlogs.git?path=/unity#main"
```
Затем `Tools > PlayJoy > FastLogs > Create Config` и задать `EndpointUrl` / `AppId` / `Token`. Запись по умолчанию выключена - включается `FastLogs.StartRecording()` (или флагом `AutoStartRecording`).

Контекст и хлебные крошки (едут с каждым отчётом):
```csharp
FastLogs.SetContext("level", "3");          // пара ключ->значение
FastLogs.SetContext("playerId", "abc");
FastLogs.Breadcrumb("Player opened shop");  // событие в катящийся буфер (опц. level)
// ...
FastLogs.ClearContext();                     // при смене сцены/сессии
```

### GameMaker
Импортировать пакет из `gamemaker/`, задать `FASTLOGS_ENDPOINT` / `FASTLOGS_APP_ID` в `scr_fastlogs_config`, создать `obj_fastlogs_controller` в первой комнате (persistent), логировать через `flog(...)`.

Контекст и хлебные крошки (едут с каждым отчётом):
```gml
fastlogs_set_context("level", "3");          // пара ключ->значение
fastlogs_set_context("playerId", "abc");
fastlogs_breadcrumb("Player opened shop");   // событие в катящийся буфер (опц. level)
// ...
fastlogs_clear_context();                     // при смене комнаты/сессии
```

### Сервер
```bash
cd server
cp .env.example .env        # BASE_URL, ADMIN_TOKEN, VIEWER_TOKEN, IP_SALT
docker compose up -d        # либо: npm install && npm start
docker compose exec fastlogs node scripts/add-app.js mygame "My Game" 30
```

## Хранение логов

По умолчанию лог живёт **30 дней**, потом удаляется (sweeper). Чтобы сохранить дольше - открой ссылку лога и нажми **Pin** (закреплённый лог не удаляется, пока закрепление не снято; снять/удалить может только админ). Нужен конкретный срок - задай `retentionDays` при отправке (макс 365).

## Приватность по умолчанию

FastLogs из коробки настроен консервативно - лишнего не уезжает.

**Что НЕ шлётся по умолчанию:**

- Сырой IP тестера: клиент не отправляет его вовсе. Для rate-limit сервер берёт IP только из соединения и хранит как `salt + sha256` (см. `IP_SALT`), а не в открытом виде.
- Чувствительные поля устройства, URL и идентификаторы (имя устройства, ссылки, идентификаторы) - под флагом, который выключен по умолчанию (Unity `IncludeSensitive=false`; GameMaker `FASTLOGS_INCLUDE_SENSITIVE=false`).

**PII в логах режется (best-effort).** Перед отправкой - и перед записью в дисковый outbox для крашей - в тексте логов, значениях контекста и текстах хлебных крошек редактируются `[redacted]`: email, IPv4, IPv6, Bearer / Authorization-токены и длинные цифровые последовательности (9+ цифр). Включено по умолчанию (Unity `ScrubPii=true`; GameMaker `FASTLOGS_SCRUB_PII=true`). Чистка одноразовая (на отправке/краше), не на горячем пути логирования.

**Как настроить (панель настроек оверлея / конфиг):**

- Включить отправку чувствительных полей - тоггл *include sensitive* (`IncludeSensitive` / `FASTLOGS_INCLUDE_SENSITIVE`).
- Выключить PII-чистку - тоггл *scrub PII* (`ScrubPii` / `FASTLOGS_SCRUB_PII`).
- Расширить набор паттернов чистки: Unity - `PiiScrubber.AddPattern(...)` до первой отправки; GameMaker - `fastlogs_redact_rules_set(...)`.

**Оговорка про best-effort.** Чистка работает по регуляркам и может перередактировать (например, длинные числа или таймстемпы попадут под правило 9+ цифр). Это осознанный компромисс в пользу приватности: лучше лишний `[redacted]`, чем утёкший токен. Полагаться на неё как на единственную меру не стоит - см. ниже про доступ и ретеншн.

## Безопасность и приватность

Логи и скриншоты могут содержать персональные данные: доступ по непубличной непредсказуемой ссылке, ограниченный ретеншн, каталог под токеном. Секреты (токены, ключи деплоя) - никогда в репозиторий (см. `.gitignore`).
